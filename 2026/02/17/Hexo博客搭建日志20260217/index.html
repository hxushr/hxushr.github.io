Hexo博客个性化DIY记录（2026.02.17）

今天终于搞定了Hexo博客的个性化修改，从最开始的默认界面，一步步改成了自己喜欢的样子，过程中踩了个小坑（CSS不生效），特意记录下来，方便以后回顾，也给和我一样的新手避坑～

今日操作目标

  1. 调大顶栏标题和导航菜单的字体，让顶栏更醒目；
  2. 把文章卡片和侧边栏改成半透明样式，告别单调纯白；
  3. 修改页面整体背景色，让半透明效果更突出；
  4. 隐藏底部的底栏（Footer），让页面更简洁；
  5. 解决之前自定义CSS不生效的问题，确保所有修改正常显示。

一、准备工作：确认文件路径（关键！）

之前写的自定义CSS一直不生效，后来发现是路径配置错了，正确步骤如下：

  1. 在Hexo博客根目录（我的是 D:\Blogs\hxushr2026），找到 source 文件夹；
  2. 在 source 里新建 css 文件夹，再在 css 里新建 custom.css 文件（所有自定义样式都写在这里）；

最终效果

  1. 顶栏：标题变大加粗，导航菜单字体适中、间距合理，手机端会自动适配；
  2. 卡片：文章卡片和侧边栏都是半透明白色，悬浮在浅灰色背景上，层次感拉满；
  3. 背景：告别单调纯白，浅灰色背景柔和不刺眼，长时间看也不疲劳；
  4. 底栏：完全隐藏，页面底部更简洁，没有多余元素。

今日踩坑总结

最开始CSS不生效，核心原因是 路径配置错误 —— 把 inject 里的CSS路径写成了 /source/css/custom.css，忽略了Hexo编译后会把source作为根目录，修改成 /css/custom.css 后就正常生效了。

新手提醒：Hexo修改样式时，路径和YAML语法（冒号后加空格）一定要注意，否则很容易出现“修改了但不生效”的问题，改完后记得执行 hexo clean 清理缓存～
